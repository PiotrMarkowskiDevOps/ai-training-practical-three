# Spec: Bootcamp Scheduler Web UI

## Overview

A local single-page web application that wraps the Python bootcamp scheduler with a modern, dark-mode UI. Users select a data file, categorise trainers, choose a solver, run the schedule, and view results in both a table and a calendar view. Built with Vue 3 + Tailwind CSS on the frontend and FastAPI (Python) on the backend — no separate Node.js server needed since the scheduler is already Python.

---

## Tech Stack

| Layer     | Technology |
|-----------|------------|
| Frontend  | Vue 3 (Composition API) + Vite |
| Styling   | Tailwind CSS v3 |
| Backend   | FastAPI (Python) — wraps `scheduler.py` directly |
| HTTP      | Axios (frontend → backend) |
| Dev       | `uvicorn` for API, `vite dev` for frontend — both run locally |

## API Endpoints

### `GET /api/files`
Returns list of available xlsx files in `data/`.

```json
["basic.xlsx", "intermediate.xlsx", "advanced.xlsx"]
```

### `GET /api/trainers?file=intermediate.xlsx`
Reads the file, returns trainer names and metadata.

```json
{
  "trainers": ["Alice Chen", "Bob Smith", "Carol Jones", "Diana Müller", "Eve Brown"],
  "has_locations": true,
  "has_weightings": true
}
```

### `POST /api/schedule`
Runs the scheduler. Request body:

```json
{
  "file": "intermediate.xlsx",
  "solver": "optimal",
  "config": {
    "experienced": ["Alice Chen", "Bob Smith", "Diana Müller"],
    "trainees": ["Carol Jones"]
  },
  "apply_bank_holidays": true
}
```

Response:

```json
{
  "schedule": [
    {
      "slot": ["2026-03-16", "2026-03-17"],
      "trainers": ["Alice Chen", "Bob Smith"],
      "location": "London"
    }
  ],
  "solver_used": "optimal",
  "total_bootcamps": 5,
  "duration_ms": 842
}
```

### `POST /api/schedule/compare`
Runs both greedy and optimal, returns both results in one response (for toggle view).

```json
{
  "greedy": { "schedule": [...], "total_bootcamps": 4 },
  "optimal": { "schedule": [...], "total_bootcamps": 5 }
}
```

## Control Panel (left column, ~320px wide)

### 1. File Selector
- Label: "Data Source"
- `<select>` dropdown listing `basic.xlsx`, `intermediate.xlsx`, `advanced.xlsx`
- On change: fetches `/api/trainers?file=...` and refreshes the trainer list below
- Styled: dark background, pink border on focus

### 2. Trainer Categorisation
- Rendered after file is selected
- Each trainer shown as a card with their name
- Three-way toggle per trainer: **Experienced** | **Uncategorised** | **Trainee**
  - Default: Uncategorised
  - Experienced: pink badge
  - Trainee: amber badge
  - Uncategorised: grey badge
- Smooth toggle animation

### 3. Options
- **Apply Bank Holidays** toggle switch (on by default)
- Label explains: "Blocks trainers on public holidays for their home country"

### 4. Solver Toggle
- Two large pill buttons side by side: **Greedy** | **Optimal**
- Active: pink background, white text
- Inactive: dark surface, grey text
- Small tooltip on Optimal: "Finds globally best solution — may take up to 30s"

### 5. Run Button
- Full width, pink background, white bold text: "Generate Schedule"
- On click: shows spinner inside button, disables all controls
- Shows elapsed time in seconds while running ("Running… 4s")
- On complete: re-enables controls, scrolls to results

---

## Results Area (right column)

### Header row
- "X bootcamps scheduled" in large text
- Greedy vs Optimal toggle (only visible after compare run):
  - Pill toggle: **Greedy (4)** | **Optimal (5)**
  - Switching instantly shows cached result — no re-run
- View toggle: **Table** | **Calendar** (right-aligned)

### Table View
Columns: **Slot** | **Trainers** | **Location**

| Slot | Trainers | Location |
|------|----------|----------|
| Mon 16 – Tue 17 Mar | Alice Chen, Bob Smith | London |
| Thu 26 – Fri 27 Mar | Diana Müller, Eve Brown | Paris |

- Slot column: formatted as "Mon 16 – Tue 17 Mar 2026", monospace font
- Trainers: each name as a small pill/chip, colour-coded by category (pink = experienced, amber = trainee, grey = uncategorised)
- Location: flag emoji + city name (🇬🇧 London, 🇫🇷 Paris, 🇳🇱 Amsterdam, 🇸🇪 Stockholm)
- Rows animate in with a staggered fade (50ms delay per row)
- Empty state: "No bootcamps could be scheduled with the current constraints"

### Calendar View
- 4-week grid (Mon–Fri columns, weeks as rows)
- Each scheduled bootcamp shown as a coloured block spanning its 2 days
- Block shows: trainer initials + location
- Blocks are pink; hover shows tooltip with full details
- Unscheduled days: empty dark cell
- Public holidays blocked by bank holidays: shown with a subtle strikethrough or "🏖" indicator

### Export Row (below results)
- Two buttons: **Export CSV** | **Export JSON**
- CSV: one row per bootcamp (date1, date2, trainer1, trainer2, location)
- JSON: raw schedule array

---

## Loading & Error States

### Spinner
- Centred in results area during solve
- Animated pink ring (Tailwind `animate-spin`)
- Text below: "Running optimal solver…" + elapsed seconds counter

### Error Toast
- Top-right corner, slides in
- Red border, dark background
- Message from API error (e.g. "No feasible solution found")
- Auto-dismisses after 5s

## Acceptance Criteria

- **AC1**: Selecting a file populates trainer list with correct names from that file
- **AC2**: Trainer category toggles (Experienced/Trainee/Uncategorised) update the config sent to the API
- **AC3**: Clicking "Generate Schedule" shows spinner, calls API, displays results on completion
- **AC4**: Table view shows correct slot dates, trainer names with category badges, and location with flag
- **AC5**: Calendar view renders a 4-week grid with bootcamp blocks on correct days
- **AC6**: Greedy/Optimal toggle instantly switches between cached results without re-running
- **AC7**: Bank holidays toggle off removes the `apply_bank_holidays` call from the request
- **AC8**: Export CSV downloads a valid CSV file with one row per bootcamp
- **AC9**: If the solver returns an empty schedule, a clear empty state message is shown
- **AC10**: The UI is fully usable at 1280px+ width; control panel and results are side by side
